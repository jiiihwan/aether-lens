# AetherLens (AI 여행 사진 정리 에이전트) 상세 기능 및 사용 설명서

AetherLens는 클라우드 서버 호스팅 비용이 **완벽히 $0**가 되는 웹 브라우저 로컬 연산 기반의 차세대 AI 사진 정리 및 자동 보정 에이전트입니다.

사용자의 귀중한 사진 데이터는 외부 서버로 일절 전송되지 않으며, 사용자의 브라우저 내부에 탑재된 고성능 WebAssembly 엔진(OpenCV.js)과 로컬 알고리즘을 사용해 흔들림 검출, 중복 제거, 명암비 보정 및 여행 스토리 앨범 생성을 단 한 번에 처리합니다.

---

## 1. 상세 기능 명세 (Detailed Features)

### 📸 1. 사진 메타데이터 로더 (`photo-loader`)
* **브라우저 드래그 앤 드롭**: 대량의 이미지 파일(JPG, PNG)을 드래그하여 브라우저에 바로 던지거나 파일 탐색기를 통해 불러옵니다.
* **로컬 EXIF 데이터 파싱**: 이미지 바이너리 파일 내부의 EXIF 메타데이터를 분석하여 촬영 시각(Date/Time)과 위치 정보(GPS Latitude/Longitude)를 외부 통신 없이 즉시 추출합니다.
* **시간순 정렬**: 파일명과 상관없이 실제 촬영된 물리적 시간순으로 이미지 파이프라인을 자동 정렬합니다.

### 👁️ 2. OpenCV.js WASM 기반 흔들림 제거 (`blur-detector`)
* **Laplacian Variance 계산**: 이미지 내부의 경계선(Edge) 대비 선명도를 수학적으로 분석하기 위해 OpenCV의 라플라시안 필터를 적용하여 분산값을 산출합니다.
* **지능적 임계치 필터링**: 사용자가 설정한 선명도 기준값 이하의 사진은 흔들려 초점이 나간 사진으로 판정하여 최종 앨범에서 자동 배제시킵니다.
* **썸네일 리사이즈 연산 최적화**: 대용량 이미지를 통째로 연산 시 발생하는 브라우저 프리징(멈춤)을 방지하기 위해, 1차 분석 시 이미지를 400px 수준의 메모리 썸네일로 축소하여 분석 속도를 10배 이상 향상시켰습니다.

### 👥 3. 지각 해싱(pHash) 기반 중복/연사 제거 (`duplicate-detector`)
* **지각 해시(Perceptual Hash) 산출**: 이미지를 8x8 Grayscale로 축소한 뒤, 전체 픽셀의 평균 밝기와 각 픽셀값을 비교해 64비트 바이너리 지각 해시를 생성합니다.
* **햄밍 거리(Hamming Distance) 비교**: 사진 간의 구조적 유사성을 비교하여 구도나 촬영 각도가 거의 같은 사진 그룹(연사 묶음)을 자동 클러스터링합니다.
* **베스트 대표컷 선정**: 묶인 그룹 내에서 흔들림 점수(Laplacian Variance)가 가장 높은 1장의 완벽한 사진만 대표작(`Representative Cut`)으로 자동 선정하고, 나머지 구도가 겹치는 사진들은 정리용 휴지통으로 분류합니다.

### 🏆 4. 로컬 미적 점수 채점 (`photo-scorer`)
* **미적 등급 평가(Aesthetic Scoring)**: 이미지 픽셀 대비도(Contrast)의 표준 편차 분포와 극단적인 명암 불균형(노출 부족/과다 상태)을 검출해 미적 점수를 계산합니다.
* **우선순위 랭킹 설정**: 선명도 점수와 조도 밸런스 점수를 종합해 0점부터 10점까지의 최종 품질 평가를 진행하며, 사용자가 세팅한 상위 비율(예: 30%) 만큼만 최종 보정 후보로 선별합니다.

### 🪄 5. 국소 적응형 대비 자동 보정 (`editor`)
* **CLAHE (Contrast Limited Adaptive Histogram Equalization)**: 전체 이미지를 국소 그리드(8x8) 구역으로 쪼개어 각 구역의 명암 히스토그램을 균일화하되, 노이즈가 과대 증폭되는 것을 방지하는 대비 제한 알고리즘을 사용합니다.
* **고화질 원본 해상도 적용**: 분석은 빠르게 썸네일로 하지만, 최종 선정된 베스트 컷의 보정은 이미지 손실을 방지하기 위해 사용자가 올린 원본 해상도 그대로 픽셀 보정을 실행하여 새로운 결과 Blob을 생성합니다.
* **화이트 밸런스 및 밝기 향상**: 그늘지거나 어둡게 나온 여행 풍경 및 인물 사진의 명암비를 자연스럽게 끌어올려 세련된 톤으로 복원합니다.

### 📖 6. 시간 기반 클러스터링 및 AI 여행 스토리북 빌드 (`album-generator`)
* **시간차 클러스터링**: 사진 촬영 간격의 공백 시간(예: 3시간)을 기준으로 여행의 이벤트(예: 1일차 오전 바닷가, 1일차 오후 식당 등)를 자동으로 분리하여 폴더 구조를 시각화합니다.
* **Gemini 2.5 Flash API 결합**: 사용자가 제공한 API Key를 활용해 각 이벤트 그룹의 촬영 시각과 정량 점수 메타데이터를 기반으로, 전체 여행 소설 대제목, 각 에피소드 줄거리 이야기, 사진별 어울리는 문학적인 캡션(Caption)을 다국어(한국어 기본)로 작문합니다. (Key가 없는 경우 로컬 룰에 기반한 표준 앨범북 생성)

### 💾 7. 브라우저 내부 ZIP 압축 다운로드 (`JSZip`)
* **서버 경유 없는 로컬 압축**: 보정이 완수된 고화질 베스트 컷 파일들을 웹 브라우저 내에서 바로 가상 압축하여 `AetherLens_enhanced_photos.zip` 파일로 로컬 디스크에 즉시 내려받도록 지원합니다.

## 2. 사용 방법 가이드 (User Manual)

### 1단계: 로컬 테스트 실행
1. 에이전트 폴더 경로에서 터미널을 열고 다음 파이썬 내장 서버 구동 명령을 입력합니다.
   ```bash
   python -m http.server 8000
   ```
2. 인터넷 브라우저 주소창에 `http://localhost:8000`을 입력해 접속합니다.

### 2단계: 에이전트 설정 구성 (좌측 패널)
1. **Gemini API Key**: 감성적인 이야기 서술과 맞춤 캡션을 인공지능이 짓게 하려면 API Key를 입력합니다. (입력 시 브라우저 내 비밀 저장 공간에만 보관됩니다.)
2. **흔들림 허용치 (Laplacian)**: 기본값은 25입니다. 값이 높을수록 미세하게 흐린 사진도 가차 없이 탈락시킵니다. (흔들린 사진이 많이 걸러지길 원하면 35~40으로 조정)
3. **중복 감도 (pHash)**: 기본값은 12입니다. 값이 작을수록 정확히 구도가 똑같아야 중복으로 잡고, 값을 높이면 조금만 비슷해도 중복 사진으로 묶어버립니다.
4. **베스트 컷 선정 비율**: 정리 완료 후 최종 앨범북 및 다운로드 대상에 포함할 사진의 비율(예: 전체 중 상위 30%)을 설정합니다.

### 3단계: 사진 분석 수행
1. 여행 사진들이 모여 있는 폴더를 통째로 선택하거나, 여러 사진 파일들을 잡고 브라우저 화면의 점선 드래그 영역(`drop-zone`)에 올려놓습니다.
2. 에이전트가 작동하기 시작하면 파이프라인 그래프의 진행 단계가 불빛으로 점등되며, 하단 터미널 로그창에 실시간으로 작업 상세 내역이 한글로 표출됩니다.

### 4단계: 결과 검토 및 소장
1. **📖 여행 앨범북**: 시간대별로 묶인 에피소드 제목과 아름다운 줄거리 텍스트, 그리고 사진별 감성 캡션을 감상할 수 있습니다.
2. **✨ 보정 갤러리**: 마우스를 사진에 올리면(Hover) 원본(Before) 상태가 나타나며, 마우스를 떼면 보정(After) 상태가 나타나 두 사진의 보정 효과 차이를 실시간으로 비교해볼 수 있습니다.
3. **🗑️ 정리된 사진 리스트**: 흔들려서 탈락한 사진 목록과 중복으로 선별 제외된 사진 목록을 확인할 수 있습니다.
4. **다운로드**: 화면 우측 상단의 `보정 완료된 베스트 컷 다운로드 (.zip)` 버튼을 누르면 압축된 보정 이미지 파일들이 바로 다운로드됩니다.

## 3. 상시 온라인 무료 배포 가이드 (Deployment Guide)

사용자의 컴퓨터가 꺼져도 외부 친구나 사용자가 URL 주소를 통해 이 서비스를 24시간 항상 이용하게 하려면 무료 정적 배포를 하시면 됩니다.

### 방법 A: Vercel을 이용한 초고속 배포 (추천)
1. [Vercel 공식 홈페이지](https://vercel.com)에 가입합니다.
2. 에이전트 프로젝트 폴더에 있는 `index.html`, `style.css`, `app.js` 파일을 본인의 GitHub 저장소(Repository)에 업로드(Push)합니다.
3. Vercel 대시보드에서 `Add New` -> `Project`를 누르고, 해당 GitHub 저장소를 연동한 뒤 **Deploy**를 클릭합니다.
4. 약 30초 내로 배포가 완료되며, `https://본인앱이름.vercel.app` 형태의 상시 접속 가능한 고유 URL이 발급됩니다. (서버 비용: $0 평생 무료)

### 방법 B: GitHub Pages를 이용한 배포
1. GitHub 저장소의 `Settings` -> `Pages` 탭으로 이동합니다.
2. Branch 소스를 `main` (혹은 `master`) 브라우저 루트(`/`)로 지정한 뒤 **Save**를 누릅니다.
3. 1~2분 뒤 `https://사용자명.github.io/저장소명/` 주소로 상시 접속 가능한 무료 웹 서비스가 배포됩니다.
